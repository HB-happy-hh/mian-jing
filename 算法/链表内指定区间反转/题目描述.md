# 链表内指定区间反转

## 题目描述

将一个节点数为 size 的链表，第 m 位置到第 n 位置之间的**区间**反转，要求时间复杂度 O(n)，空间复杂度 O(1)。

**示例：**
```
输入：1 → 2 → 3 → 4 → 5 → NULL，m = 2，n = 4
返回：1 → 4 → 3 → 2 → 5 → NULL
```

**数据范围：**
- 链表长度 0 < size ≤ 1000
- 0 < m ≤ n ≤ size
- 链表中每个节点的值满足 |val| ≤ 1000

**要求：**
- 普通解法：时间复杂度 O(n)，空间复杂度 O(n)
- 进阶解法：时间复杂度 O(n)，空间复杂度 O(1)

---

## 解题思路：头插法（O(1) 空间）

**核心思想：** 找到第 m 个节点的前驱 `pre`，将 `cur`（第 m 个节点）之后的节点依次"头插"到 `pre` 后面，重复 `n - m` 次。

### 指针含义

| 指针 | 含义 | 是否移动 |
|------|------|---------|
| `dummy` | 哑节点，避免处理头节点特殊情况 | 不动 |
| `pre` | 反转区间的前驱节点 | 不动（定位后固定） |
| `cur` | 反转区间的第一个节点（第 m 个） | **不动**（始终是 cur） |
| `next` | cur 后面待头插的节点 | 每次取新的 |

### 每次头插的 4 个操作

```
操作前：... → pre → [cur → next → ...]
                              ↓
① next = cur.next          取出 next
② cur.next = next.next     cur 跳过 next
③ next.next = pre.next     next 指向旧的反转区第一个
④ pre.next = next          pre 指向 next（头插完成）
```

### 执行过程示意（1→2→3→4→5→6→7，m=2，n=6）

```
初始：  dummy→1→[2→3→4→5→6]→7    pre=node(1)，cur=node(2)

第1次：把 node(3) 头插到 pre 后
       dummy→1→[3→2→4→5→6]→7

第2次：把 node(4) 头插到 pre 后
       dummy→1→[4→3→2→5→6]→7

第3次：把 node(5) 头插到 pre 后
       dummy→1→[5→4→3→2→6]→7

第4次：把 node(6) 头插到 pre 后
       dummy→1→[6→5→4→3→2]→7

结果：  1 → 6 → 5 → 4 → 3 → 2 → 7 → NULL  ✓
```

**规律：** `cur`（node(2)）始终固定不动，每次把它后面的节点摘下来插到 `pre` 正后方，新节点每次都排在最前面，天然实现逆序。

## Java 代码（核心代码模式）

```java
public ListNode reverseBetween(ListNode head, int m, int n) {
    ListNode dummy = new ListNode(0);
    dummy.next = head;
    ListNode pre = dummy;

    // pre 移到第 m 个节点的前一个
    for (int i = 1; i < m; i++) {
        pre = pre.next;
    }

    ListNode cur = pre.next; // cur 固定在第 m 个节点，不再移动

    // 头插法：将 cur 后面的节点依次插到 pre 后面，共 n-m 次
    for (int i = 0; i < n - m; i++) {
        ListNode next = cur.next;   // ① 取出 cur 的下一个节点
        cur.next = next.next;       // ② cur 跳过 next
        next.next = pre.next;       // ③ next 指向原来 pre 的下一个（旧的第一个）
        pre.next = next;            // ④ pre 的下一个改为 next（头插完成）
    }

    return dummy.next;
}
```

## 复杂度分析

| | 时间复杂度 | 空间复杂度 |
|---|---|---|
| 本解法（头插法） | O(n) | **O(1)** |
| 暴力（存数组再重建） | O(n) | O(n) |

---

## 常见错误

| 错误 | 原因 |
|------|------|
| `return cur` 而非 `return pre` | 循环结束时 cur 为 null |
| 忘记使用 `dummy` 节点 | m=1 时 pre 无前驱，容易 NPE |
| `cur` 跟着移动 | cur 必须固定在第 m 个节点，是每次头插的"锚点" |
