StringBuilder 图解

String 不可变，每次拼接都产生新对象；StringBuilder 内部维护一块可扩展的字符数组，
原地修改，零拷贝，是高频字符串操作的首选。

String vs StringBuilder

String — 不可变

每次 + 都新建一个对象，旧对象等待 GC 回收。
循环拼接 n 次 → 产生 n 个临时对象，O(n²) 时间。

String s = "";
s = s + "(";   // 新对象 "("
s = s + ")";   // 新对象 "()"，旧的丢弃
// 每步都复制整个字符串！

StringBuilder — 可变

内部维护 char[]，append 直接写入，容量不够才扩容（×2）。
循环拼接 n 次 → O(n) 时间，O(1) 均摊每次操作。

StringBuilder sb = new StringBuilder();
sb.append('(');  // 原地写入，无新对象
sb.append(')');  // 继续写入同一块内存
String s = sb.toString(); // 最后才生成 String

常用 API 速查
方法返回值说明
append(x)StringBuilder追加任意类型（char/String/int…）到末尾，返回 this（支持链式调用）
deleteCharAt(i)StringBuilder删除索引 i 处的字符，回溯撤销时常用
delete(s,e)StringBuilder删除 [s, e) 范围内的字符
insert(i,x)StringBuilder在索引 i 处插入字符/字符串
reverse()StringBuilder原地反转所有字符
charAt(i)char获取索引 i 处的字符，O(1)
setCharAt(i,c)void修改索引 i 处的字符
length()int当前字符数（≠ 内部数组容量）
toString()String将缓冲区内容转为不可变 String，通常在最终收集结果时调用
indexOf(s)int返回子串第一次出现的位置，-1 表示不存在
substring(s,e)String截取 [s, e) 的子串（不修改原对象）
append 链式：sb.append('(').append(')')
初始容量默认 16，不够时自动扩容 ×2+2
线程不安全；多线程场景用 StringBuffer
