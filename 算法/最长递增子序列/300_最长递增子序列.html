LeetCode #300

最长递增子序列

中等

给你一个整数数组 nums，找到其中最长严格递增子序列的长度。子序列不要求连续，但顺序不能改变。

核心思路：动态规划
1
定义状态：dp[i] = 以 nums[i] 结尾的最长严格递增子序列的长度。 初始全为 1（每个元素自身就是长度 1 的子序列）。
2
状态转移：对每个 i，向左扫描所有 j < i：
  若 nums[j] < nums[i]，则 nums[i] 可以接在 nums[j] 后面
  → dp[i] = max(dp[i], dp[j] + 1)
3
答案：所有 dp[i] 中的最大值（不一定是 dp[n-1]，因为最长子序列不一定以最后一个元素结尾）。
关键洞察
!
dp[i] 只依赖它左边的所有状态，因此从左到右填表，每填一格就已知它之前所有结果。 两层循环，外层 i 从 1 到 n-1，内层 j 从 0 到 i-1。

Java 代码实现
300_LIS.java 
import java.util.*;

/**
 * LeetCode 300. 最长递增子序列
 * 时间：O(n²)  空间：O(n)
 */
class Solution {
    public int lengthOfLIS(int[] nums) {
        int n = nums.length;

        // dp[i]：以 nums[i] 结尾的最长递增子序列长度，初始都为 1
        int[] dp = new int[n];
        Arrays.fill(dp, 1);

        int ans = 1;

        for (int i = 1; i < n; i++) {
            // 向左扫描所有 j < i 的位置
            for (int j = 0; j < i; j++) {
                if (nums[j] < nums[i]) {
                    // nums[i] 可以接在 nums[j] 后面，更新 dp[i]
                    dp[i] = Math.max(dp[i], dp[j] + 1);
                }
            }
            ans = Math.max(ans, dp[i]);
        }

        return ans;
    }
}
复杂度分析
维度复杂度说明
时间O(n²)两层循环，外层 n 次，内层最多 n 次
空间O(n)dp 数组长度为 n
易错点
⚠
答案不是 dp[n-1]，而是 max(dp[0..n-1])。 最长子序列可能以任何位置的元素结尾。
⚠
严格递增：条件是 nums[j] < nums[i]，相等不算。 示例3 全为 7，所以答案是 1。
⚠
子序列 ≠ 子数组：子序列可以跳过元素，只要保持原来的相对顺序即可。 比如 [10,2,5,7,101] 中 [2,5,7] 是子序列。
