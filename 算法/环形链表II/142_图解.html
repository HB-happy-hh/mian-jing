环形链表 II

给定链表头节点，返回链表开始入环的第一个节点，无环返回 null。
核心：Floyd 判圈算法 —— 数学推导出"相遇点到入环点"的距离恰好等于"head 到入环点"的距离。

数学推导：为什么第二阶段能找到入环点

定义变量

a = head 到入环点的距离
b = 入环点到相遇点的距离
c = 相遇点沿环回到入环点的距离
环长 = b + c

第一阶段相遇时

slow 走了：a + b
fast 走了：a + b + k(b+c)
fast = 2 × slow：
2(a+b) = a+b+k(b+c)

a + b = k(b+c)
a = k(b+c) - b
a = (k-1)(b+c) + c

关键结论（k=1）

a = c

head 到入环点 = 相遇点到入环点

所以：一个指针从 head 出发，一个从相遇点出发，同速走，必在入环点相遇！

代码实现
DetectCycle.java 
public ListNode detectCycle(ListNode head) {
    ListNode slow = head, fast = head;

    // 第一阶段：判断是否有环，找相遇点
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) {
            // 第二阶段：找入环点（a = c）
            ListNode ptr = head;
            while (ptr != slow) {
                ptr  = ptr.next;
                slow = slow.next;
            }
            return ptr;  // 入环点
        }
    }
    return null;  // 无环
}
public ListNode detectCycleHashSet(ListNode head) {
    Set<ListNode> visited = new HashSet<>();
    ListNode cur = head;
    while (cur != null) {
        if (visited.contains(cur)) return cur; // 首次重复 = 入环点
        visited.add(cur);
        cur = cur.next;
    }
    return null;
}
复杂度分析
方法时间空间说明
Floyd 快慢指针O(n)O(1)两个指针，无额外空间，面试首选
HashSetO(n)O(n)直观易理解，但空间换时间
fast = slow 时必在环内（fast 比 slow 多走整数圈）
a = c 是数学必然，与链表长度、环位置无关
