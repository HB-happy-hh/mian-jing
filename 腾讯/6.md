
（2）介绍一下垃圾回收算法。
答：垃圾回收的三种基础算法：

**① 标记-清除（Mark-Sweep）**
分两阶段：标记阶段从 GC Roots 出发遍历所有可达对象并打标记；清除阶段扫描整个堆，回收未被标记的对象。
缺点：会产生大量内存碎片，分配大对象时可能触发 Full GC。

**② 标记-复制（Mark-Copy）**
将堆分为两块等大的空间（From/To），每次只使用 From 区。GC 时把 From 区存活对象全部复制到 To 区，然后清空 From 区，两块区域角色互换。
优点：没有碎片，分配速度快（指针碰撞）。
缺点：内存利用率只有 50%。JVM 新生代的 Eden + 两个 Survivor 区就是对这种思路的优化（实际浪费只有 1/10）。

**③ 标记-整理（Mark-Compact）**
标记阶段同标记-清除；整理阶段将所有存活对象向一端移动，然后直接清掉边界以外的内存。
优点：没有碎片，内存利用率高。
缺点：移动对象需要更新引用，停顿时间比标记-清除长。适合老年代（CMS 的 Full GC、G1 的 mixed GC 都会用到）。

**实际 JVM 的分代策略**：新生代对象存活率低，用复制算法效率高；老年代对象存活率高，用标记-整理减少碎片。现代 GC（G1、ZGC）在此基础上做了并发化和分区化优化，进一步降低 STW 停顿时间。
（3）了解过抓包吗？介绍一下tcpdump？
答：tcpdump 是 Linux 下的命令行抓包工具，工作在网络层，通过 libpcap 库捕获经过网卡的数据包，可以输出到终端或保存为 .pcap 文件供 Wireshark 分析。

**常用参数：**
- `-i eth0`：指定监听的网卡，`-i any` 监听所有网卡
- `host 192.168.1.1`：只抓与指定 IP 相关的包
- `port 8080`：只抓指定端口的包
- `-n`：不做 DNS 反向解析，输出更快
- `-w out.pcap`：抓包结果写入文件
- `-r out.pcap`：读取已保存的文件进行分析

**常用组合示例：**
```bash
# 抓取与某个 IP 的 80 端口通信
tcpdump -i eth0 host 10.0.0.1 and port 80 -n

# 抓 TCP 三次握手（只看 SYN/SYN-ACK/ACK）
tcpdump -i eth0 'tcp[tcpflags] & (tcp-syn|tcp-ack) != 0'

# 保存抓包结果
tcpdump -i eth0 port 443 -w https.pcap
```

**实际用途：**
1. 排查请求是否到达服务器（看有没有对应的包进来）
2. 分析 TCP 握手/挥手是否正常
3. 排查网络延迟或丢包问题
4. 配合 Wireshark 分析 HTTP/HTTPS 报文内容

与 Wireshark 的区别：tcpdump 是纯命令行，适合在没有 GUI 的服务器上使用；Wireshark 有图形界面，分析更直观，两者可以配合使用（tcpdump 抓包，Wireshark 分析）。
（4）介绍一下分布式系统的一致性哈希算法。
答：一致性哈希算法主要解决数据如何在多台机器之间分布的问题，主要方法是首先计算节点的hash值，然后计算数据的hash值，把分布式节点和数据的hash都映射到一个数据环上，然后从数据的位置开始，顺时针找到的第一个分布式节点的位置就是存储数据的位置。比如说数据环的范围是0-2^32，某个数据的hash的值是150，从150顺时针找，比如找到第一个分布式节点的位置是200，那就由这个节点存储。这样做的优点是在节点增减时，只影响少量数据，避免全量重分布；并且结合虚拟节点实现，数据分布更均匀。
（5）介绍一下HTTPS四次握手。
答：1. ClientHello（第一次握手）：客户端（例如浏览器）向服务器发送握手请求，包含TLS版本，加密套件（对称加密、非对称加密算法等）和随机数（Client Random）。
2. ServerHello（第二次握手）：服务器选择加密算法，生成随机数（Server Random），随着数字证书一起返回。
3. 密钥交换（第三次握手）：客户端验证数字证书有效性，然后生成第三个随机数（pre-master key，预主密钥），使用公钥进行加密，返回给服务器公钥加密的预主密钥。此时利用协商的对称加密算法，生成对称密钥，保存在本地。
4. 握手完成（第四次握手）：服务器通过私钥解密，拿到预主密钥，同样使用对称加密算法计算出对称密钥。
（6）算法题：实现两个字符串类型的大数相加。（ACM风格）
答：从两个字符串的末尾同时向左遍历，逐位相加并处理进位，结果逆序输出。

思路：
- 用两个指针 i、j 分别指向 a、b 的末位，carry 保存进位。
- 每轮：sum = carry + (a[i] 的数字，若有) + (b[j] 的数字，若有)
- 本位结果 = sum % 10，新进位 = sum / 10
- 循环直到两个指针都越界且 carry = 0 为止
- 最终对结果字符串反转即可

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        String a = sc.next();
        String b = sc.next();
        System.out.println(addStrings(a, b));
    }

    public static String addStrings(String a, String b) {
        StringBuilder sb = new StringBuilder();
        int i = a.length() - 1;
        int j = b.length() - 1;
        int carry = 0;

        while (i >= 0 || j >= 0 || carry > 0) {
            int sum = carry;
            if (i >= 0) sum += a.charAt(i--) - '0';
            if (j >= 0) sum += b.charAt(j--) - '0';
            carry = sum / 10;
            sb.append(sum % 10);
        }

        return sb.reverse().toString();
    }
}
```

示例：
输入：9999999999  1
输出：10000000000

时间复杂度：O(max(m, n))，m/n 为两字符串长度
空间复杂度：O(max(m, n))，结果字符串长度
