Q6. 详细解释进程、线程和协程的区别？哪一个的上下文切换开销最小？
进程（Process）：资源分配的最小单位
本质：进程是操作系统进行资源分配和调度的基本单位。你可以把它理解为一个正在运行的程序的实例（比如你打开了一个 JVM 进程，或者打开了一个微信）。

特点：每个进程都有自己完全独立的内存空间（包括代码段、数据段、堆区等）。这种强隔离性意味着一个进程崩溃通常不会影响其他进程，系统的稳定性很高。

痛点：正因为资源独立，进程的创建、销毁以及上下文切换的开销极其巨大。因为切换进程需要保存和恢复大量的寄存器状态、内存页表等信息，这涉及到从用户态到内核态的频繁陷入。

2. 线程（Thread）：CPU 调度的最小单位
本质：线程是进程内部的一个执行流，是 CPU 实际调度和分派的基本单位。

特点：一个进程可以包含多个线程，这些线程共享该进程的内存空间（比如 Java 中的堆和方法区），但每个线程有自己独立的虚拟机栈、本地方法栈和程序计数器。

痛点：

虽然线程的切换开销比进程小很多，但传统线程（在 Linux 和 Java 早期版本中）仍然和操作系统的内核线程是 1:1 绑定的。因此，线程的挂起、唤醒和切换依然需要操作系统的内核介入，这同样涉及内核态切换，开销不容忽视。

因为共享内存，多线程并发时必然面临线程安全问题，我们不得不大量使用锁（如 synchronized、ReentrantLock），这又进一步加剧了线程的阻塞和上下文切换开销。

3. 协程（Coroutine）：用户态的“轻量级线程”
本质：协程不是由操作系统内核管理的，而是完全由用户态的应用程序（或语言的运行时环境 Runtime）自己来调度和管理的执行流。

特点：

极度轻量级：一个线程动辄占用 1MB 的内存栈，而一个协程通常只需要几 KB。因此，在一台机器上创建几十个线程可能就面临瓶颈，但创建上百万个协程却毫无压力。

无内核切换开销：协程的挂起和恢复只在用户态进行保存和恢复寄存器状态，不需要陷入操作系统内核，切换速度是微秒甚至纳秒级的。

协作式调度：线程往往是由 CPU 抢占式调度的，而协程通常是协作式的，一个协程主动让出（Yield）执行权，其他协程才能执行。
