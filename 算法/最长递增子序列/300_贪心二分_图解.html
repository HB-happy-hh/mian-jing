LeetCode #300

最长递增子序列

中等 方法二：贪心 + 二分查找

在 O(n log n) 时间内求解，核心是维护一个「最优末尾」数组 tails，保证每种长度的子序列末尾值尽可能小。

贪心核心思想
维护数组 tails，其中 tails[i] 表示长度为 i+1 的所有递增子序列中，末尾元素的最小值。

为什么要让末尾尽量小？末尾越小，后面来的新数字越容易比它大，从而接在后面让子序列变长。这就是「贪心」的含义。

对每个新数字 num，二分查找 tails 中第一个 ≥ num 的位置 lo，然后令 tails[lo] = num：
  • lo == tails 长度 → num 比所有末尾都大，追加，LIS 长度 +1
  • lo < tails 长度 → 替换 tails[lo]，让长度为 lo+1 的子序列末尾变得更小

Java 代码
Solution.java — 贪心 + 二分 O(n log n) 
class Solution {
    public int lengthOfLIS(int[] nums) {
        // tails[i] = 长度为 i+1 的递增子序列的最小末尾值
        int[] tails = new int[nums.length];
        int len = 0;   // tails 的有效长度，即当前 LIS 长度

        for (int num : nums) {
            // 二分查找：在 tails[0..len-1] 中找第一个 >= num 的位置
            int lo = 0, hi = len;
            while (lo < hi) {
                int mid = (lo + hi) / 2;
                if (tails[mid] < num) lo = mid + 1;
                else                  hi = mid;
            }

            // 替换（让那个长度的末尾值更小）或追加（LIS 变长）
            tails[lo] = num;
            if (lo == len) len++;
        }

        return len;
    }
}
二分查找的目标：找第一个 ≥ num 的位置
为什么找 ≥ 而不是 >？
严格递增要求不能有相等元素。若 num == tails[lo]，用 num 替换 tails[lo] 不改变值但保持正确性； 若找 >，相等时会追加，导致 tails 中出现重复，破坏严格递增的语义。
