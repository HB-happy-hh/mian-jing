# 23. 合并 K 个升序链表

**难度**：困难

## 题目描述

给你一个链表数组，每个链表都已经按升序排列。

请你将所有链表合并到一个升序链表中，返回合并后的链表。

## 示例

**示例 1：**
```
输入：lists = [[1,4,5],[1,3,4],[2,6]]
输出：[1,1,2,3,4,4,5,6]
解释：链表数组如下：
[
  1->4->5,
  1->3->4,
  2->6
]
将它们合并到一个有序链表中得到：
1->1->2->3->4->4->5->6
```

**示例 2：**
```
输入：lists = []
输出：[]
```

**示例 3：**
```
输入：lists = [[]]
输出：[]
```

## 提示

- `k == lists.length`
- `0 <= k <= 10⁴`
- `0 <= lists[i].length <= 500`
- `-10⁴ <= lists[i][j] <= 10⁴`
- `lists[i]` 按**升序**排列
- `lists[i].length` 的总和不超过 `10⁴`

## 思路

使用**最小堆（优先队列）** 解决：

- 将 k 个链表的头节点全部入堆，堆按节点值从小到大排序。
- 每次从堆顶取出值最小的节点，接到结果链表末尾。
- 若该节点有后继节点，将后继节点入堆，继续参与竞争。
- 重复直到堆为空。

```
初始堆：[1(L1), 1(L2), 2(L3)]   → 取出 1(L1)，将 4(L1) 入堆
堆：    [1(L2), 2(L3), 4(L1)]   → 取出 1(L2)，将 3(L2) 入堆
堆：    [2(L3), 3(L2), 4(L1)]   → 取出 2(L3)，将 6(L3) 入堆
...
结果：  1 → 1 → 2 → 3 → 4 → 4 → 5 → 6
```

**时间复杂度**：O(N log k)，N 为所有节点总数，每次堆操作 O(log k)  
**空间复杂度**：O(k)，堆中最多同时存放 k 个节点
