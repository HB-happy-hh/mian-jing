LeetCode #56

合并区间

中等

以数组 intervals 表示若干个区间的集合，合并所有重叠的区间，返回一个不重叠的区间数组，恰好覆盖输入中的所有区间。

核心思路：排序 + 贪心合并
1
排序：按照每个区间的起始值升序排列。排序后，能重叠的区间一定是相邻的。
2
初始化：取第一个区间作为"当前区间" cur，加入结果列表。
3
遍历剩余区间：对于每个区间 next，判断：
   • 若 next[0] <= cur[1]（有重叠或相邻）→ 合并：cur[1] = max(cur[1], next[1])
   • 否则 → 无重叠，将 next 作为新的 cur 加入结果
4
返回结果列表，转为二维数组。
关键洞察
!
排序后，若区间 A 的起点 ≤ 区间 B 的起点，则只需检查 A 的终点是否 ≥ B 的起点。若是，则合并后新区间终点 = max(A终点, B终点)（处理 B 被 A 完全包含的情况）。

Java 代码实现
import java.util.*;

/**
 * LeetCode 56. 合并区间
 * 时间：O(n log n)  空间：O(n)
 */
class Solution {
    public int[][] merge(int[][] intervals) {
        Arrays.sort(intervals, (a, b) -> a[0] - b[0]);

        List<int[]> result = new ArrayList<>();

        for (int[] cur : intervals) {
            int[] last = result.isEmpty() ? null : result.get(result.size() - 1);

            if (last == null || last[1] < cur[0]) {
                result.add(cur);                          // 无重叠，直接加入
            } else {
                last[1] = Math.max(last[1], cur[1]);  // 有重叠，扩展终点
            }
        }

        return result.toArray(new int[0][]);
    }
}
import java.util.*;

/**
 * LeetCode 56. 合并区间
 *
 * 思路：排序 + 贪心合并
 *  1. 按区间起始值升序排序，确保能重叠的区间一定相邻
 *  2. 遍历排序后的区间，与结果列表末尾区间比较：
 *     - 有重叠（next[0] <= cur[1]）→ 合并，终点取 max
 *     - 无重叠                     → 直接追加到结果
 *
 * 时间复杂度：O(n log n)  排序主导
 * 空间复杂度：O(n)        结果列表
 */
class Solution {
    public int[][] merge(int[][] intervals) {
        Arrays.sort(intervals, (a, b) -> a[0] - b[0]);

        List<int[]> result = new ArrayList<>();

        for (int[] cur : intervals) {
            int[] last = result.isEmpty() ? null : result.get(result.size() - 1);

            if (last == null || last[1] < cur[0]) {
                result.add(cur);
            } else {
                last[1] = Math.max(last[1], cur[1]);
            }
        }

        return result.toArray(new int[0][]);
    }
}

// ─── 测试 ────────────────────────────────────────────
class Main {
    public static void main(String[] args) {
        Solution sol = new Solution();

        // 示例1：[[1,3],[2,6],[8,10],[15,18]] → [[1,6],[8,10],[15,18]]
        print(sol.merge(new int[][]{{1,3},{2,6},{8,10},{15,18}}));

        // 示例2：[[1,4],[4,5]] → [[1,5]]（端点相邻也算重叠）
        print(sol.merge(new int[][]{{1,4},{4,5}}));

        // 示例3：[[4,7],[1,4]] → [[1,7]]（输入未排序）
        print(sol.merge(new int[][]{{4,7},{1,4}}));

        // 嵌套：[[1,10],[2,3]] → [[1,10]]（后区间被完全包含）
        print(sol.merge(new int[][]{{1,10},{2,3}}));
    }

    static void print(int[][] res) {
        StringBuilder sb = new StringBuilder("[");
        for (int i = 0; i < res.length; i++) {
            sb.append(Arrays.toString(res[i]));
            if (i < res.length - 1) sb.append(", ");
        }
        sb.append("]");
        System.out.println(sb);
    }
}
复杂度分析
维度复杂度说明
时间O(n log n)排序主导；后续遍历为 O(n)
空间O(n)结果列表最多存 n 个区间
易错边界
⚠
区间相邻（端点相等）也算重叠：[1,4] 和 [4,5] 合并为 [1,5]， 判断条件用 < 而非 <=。
⚠
后一个区间被前一个完全包含：[1,10] 和 [2,3]， 合并终点必须取 max，不能直接赋值。
⚠
输入未排序：示例 3 中 [[4,7],[1,4]]，排序后才能正确合并。
