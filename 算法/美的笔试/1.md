### 题目1：下列程序执行结果正确的是？

```java
int i = 0;
System.out.println(++i);
System.out.println(i++);
System.out.println(i +=1);
```

- A. 2,2,3
- B. 1,2,3
- C. 1,1,3
- D. 0,1,2

**答案：C（1,1,3）**

**解析：**

| 语句 | 说明 | i 的变化 | 打印值 |
|------|------|---------|--------|
| `++i` | 前置自增：先加再用 | 0 → 1 | **1** |
| `i++` | 后置自增：先用再加 | 1 → 2 | **1** |
| `i+=1` | i = i + 1 | 2 → 3 | **3** |

- `++i`（前置）：先将 i 加 1，再返回 i 的值 → 打印 1
- `i++`（后置）：先返回 i 的当前值，再将 i 加 1 → 打印 1，之后 i 变为 2
- `i+=1`：i = 2 + 1 = 3，打印 3

### 题目2：Internet上一个B类网络的子网掩码为255.255.252.0，则理论上每个子网的主机数最多可以有多少台？

- A. 4096
- B. 1024
- C. 2048
- D. 1022

**答案：D（1022）**

**解析：**

将子网掩码 `255.255.252.0` 转为二进制：

```
255 = 11111111
255 = 11111111
252 = 11111100
  0 = 00000000
```

完整二进制：`11111111.11111111.11111100.00000000`

**主机位** = 末尾连续 0 的个数 = 后2位（252的末2位）+ 8位（最后一组）= **10位**

**可用主机数** = 2^10 - 2 = 1024 - 2 = **1022**

> 减 2 原因：
> - 全0 → 网络地址（不能分配给主机）
> - 全1 → 广播地址（不能分配给主机）

### 题目3：在Java网络编程中，使用客户端套接字Socket创建对象时，需要指定？

- A. 服务器地址和文件
- B. 服务器端口和文件
- C. 服务器名称和文件
- D. 服务器主机名和端口

**答案：D（服务器主机名和端口）**

**解析：**

在 Java 中，客户端 `Socket` 的构造方法：

```java
Socket socket = new Socket("192.168.1.1", 8080);
//                          ↑ 服务器主机名/IP    ↑ 端口号
```

| 构造方法 | 说明 |
|---------|------|
| `Socket(String host, int port)` | 指定主机名/IP 和端口 |

创建客户端 Socket 必须知道：
- **连接谁**：服务器主机名或 IP 地址
- **连哪个服务**：端口号

两者缺一不可。

### 题目4：下列哪些集合类是线程安全的？

- A. Vector
- B. ConcurrentHashMap
- C. Stack
- D. HashTable

**答案：A、B、C、D 全部是线程安全的**

**解析：**

| 类 | 线程安全 | 说明 |
|----|---------|------|
| `Vector` | ✅ | 所有方法用 `synchronized` 修饰，遗留类，性能低 |
| `ConcurrentHashMap` | ✅ | JUC 包，JDK8 用 CAS + synchronized 锁链表头，性能高 |
| `Stack` | ✅ | 继承自 `Vector`，遗留类 |
| `HashTable` | ✅ | 所有方法用 `synchronized` 修饰，遗留类，性能低 |

**区别：**
- `Vector`、`Stack`、`HashTable`：早期遗留类，对整个方法加锁，**锁粒度大，性能差**
- `ConcurrentHashMap`：JUC 专为并发设计，**锁粒度细，性能好**

**实际开发推荐：**
- 用 `ConcurrentHashMap` 替代 `HashTable`
- 用 `CopyOnWriteArrayList` 或 `Collections.synchronizedList()` 替代 `Vector`

### 题目5：序列插入后的最长非递减子序列

**题目描述：**

给定两个长度为 n 的序列 a、b，顺序遍历序列 b 中的每一个元素，依次将其插入到序列 a 中，但每次插入需保证当前数字所插入的位置在之前插入的数字的位置之后（即序列 b 中的元素插入到 a 中相对顺序不变）。询问所有不同的序列 a 中最长非递减子序列最大的长度。

**输入格式：**
```
第一行：整数 n
第二行：序列 a 的 n 个元素
第三行：序列 b 的 n 个元素
```

**输出格式：**
```
一个整数，表示最长非递减子序列的最大长度
```

**示例：**
```
输入：
3
3 1 2
2 3 1

输出：
4
```

---

**解题思路：**

**结论：答案 = LNDS(a) + LNDS(b)**

其中 LNDS 表示最长非递减子序列（Longest Non-Decreasing Subsequence）。

**证明：**

- **上界**：任意插入方案得到的 2n 长度序列中，其最长非递减子序列里，a 的元素构成 a 的非递减子序列，b 的元素构成 b 的非递减子序列，故总长度 ≤ LNDS(a) + LNDS(b)。

- **下界可达**：取 a 的最长非递减子序列 A' 和 b 的最长非递减子序列 B'，将它们像归并排序一样合并成一个非递减序列，这对应一种合法的插入方案（两者相对顺序均保持），可以得到长度为 |A'|+|B'| 的非递减子序列。

**验证示例** `a=[3,1,2], b=[2,3,1]`：
- LNDS(a) = 2，取 [1,2]
- LNDS(b) = 2，取 [2,3]
- 合并 [1,2] 和 [2,3] → 插入方案 [3,1,2,2,3,1]，非递减子序列 [1,2,2,3]，长度 4 ✓

**LNDS 算法（O(n log n)）：**

维护数组 `tails`，`tails[i]` 表示长度为 `i+1` 的非递减子序列末尾元素的最小值。
- 对每个元素 x，用 `upper_bound` 找第一个 `> x` 的位置替换（找不到则追加）
- 最终 `tails` 的长度即为 LNDS 长度

> 区别：最长严格递增子序列（LIS）用 `lower_bound`，非递减用 `upper_bound`

**Java 代码：**

```java
import java.util.*;

public class Main {

    static int lnds(int[] arr) {
        int[] tails = new int[arr.length];
        int size = 0;
        for (int x : arr) {
            // 非递减用 upper_bound：找第一个严格大于 x 的位置
            int lo = 0, hi = size;
            while (lo < hi) {
                int mid = (lo + hi) / 2;
                if (tails[mid] <= x) lo = mid + 1;
                else hi = mid;
            }
            tails[lo] = x;
            if (lo == size) size++;
        }
        return size;
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();

        int[] a = new int[n];
        int[] b = new int[n];
        for (int i = 0; i < n; i++) a[i] = sc.nextInt();
        for (int i = 0; i < n; i++) b[i] = sc.nextInt();

        System.out.println(lnds(a) + lnds(b));
    }
}
```

**复杂度：** 时间 O(n log n)，空间 O(n)

### 题目6：输出螺旋三角形图形

**题目描述：**

输入一个 n，判断能否输出 n 对应的图形。如果不行输出 no；如果可以先输出一行 yes，然后再输出图形。

规律（以 n=10 为例）：
```
01
02 09
03 10 08
04 05 06 07
```

**输入格式：** 一个整数 n，1 ≤ n ≤ 60

**输出格式：** 若 n 能构造对应图形，先输出 yes 再输出图形，否则输出 no

**示例：**
```
输入：10
输出：
yes
01
02 09
03 10 08
04 05 06 07
```

---

**解题思路：**

**判断合法性：** n 必须是三角数，即存在整数 k 使得 n = k*(k+1)/2。

**填充规律（螺旋填充）：** 对 k 行的三角形，按如下顺序依次填入数字 1, 2, ..., n：
1. **左边**（从上到下）：填 k 个数
2. **底边**（从左到右，跳过左角）：填 k-1 个数
3. **右斜边**（从下到上，跳过两角）：填 k-2 个数
4. **内层三角**（大小 k-3，起点偏移 +2行 +1列）：递归填充

以 k=4，n=10 为例：
- 左边：位置 (1,1)(2,1)(3,1)(4,1) → 值 1,2,3,4
- 底边：位置 (4,2)(4,3)(4,4) → 值 5,6,7
- 右斜边：位置 (3,3)(2,2) → 值 8,9
- 内层（k=1）：位置 (3,2) → 值 10

**Java 代码：**

```java
import java.util.*;

public class Main {
    static int[][] grid = new int[65][65];
    static int num = 1;

    static void fill(int startR, int startC, int k) {
        if (k <= 0) return;
        // 左边：从上到下
        for (int i = 0; i < k; i++) {
            grid[startR + i][startC] = num++;
        }
        // 底边：从左到右（跳过左角）
        for (int i = 1; i < k; i++) {
            grid[startR + k - 1][startC + i] = num++;
        }
        // 右斜边：从下到上（跳过两角）
        for (int i = k - 2; i >= 1; i--) {
            grid[startR + i][startC + i] = num++;
        }
        // 递归填内层三角（大小 k-3，起点偏移 +2行 +1列）
        fill(startR + 2, startC + 1, k - 3);
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();

        // 判断 n 是否为三角数
        int k = 0;
        while (k * (k + 1) / 2 < n) k++;
        if (k * (k + 1) / 2 != n) {
            System.out.println("no");
            return;
        }

        System.out.println("yes");
        fill(1, 1, k);

        StringBuilder sb = new StringBuilder();
        for (int r = 1; r <= k; r++) {
            for (int c = 1; c <= r; c++) {
                if (c > 1) sb.append(' ');
                sb.append(String.format("%02d", grid[r][c]));
            }
            sb.append('\n');
        }
        System.out.print(sb);
    }
}
```

**边界情况验证：**

| n | k | 是否合法 |
|---|---|---------|
| 1 | 1 | yes |
| 3 | 2 | yes |
| 6 | 3 | yes |
| 10 | 4 | yes |
| 5 | - | no |
| 55 | 10 | yes |

### 题目7：数组最大值与首位交换，最小值与末位交换

**题目描述：**

输入数组，以空格作为分隔。最大的与第一个元素交换，最小的与最后一个元素交换，输出数组。

注意：
- 找到该数组最大项把它和第一位交换
- 找到该数值的最小项把它和最后一项交换
- 没叫到要交换的数值的位置不变（即第一步交换后，在更新后的数组中查找最小值）

**示例：**
```
输入：1 6 4 3 7 5
输出：7 6 4 3 5 1
```

---

**解题思路：**

1. 找最大值下标 `maxIdx`，与 `arr[0]` 交换
2. 在**交换后的数组**中找最小值下标 `minIdx`，与 `arr[n-1]` 交换
3. 若最大值已在首位 / 最小值已在末位，则无需交换

**关键点：** 第一步交换后，原来位于首位的元素移到了最大值原来的位置。第二步要在更新后的数组中查找最小值（例如原来的最小值在首位，交换后它移到了最大值原来的位置，要在新位置找到它）。

**验证示例** `1 6 4 3 7 5`：

| 步骤 | 操作 | 数组状态 |
|------|------|---------|
| 初始 | - | `[1, 6, 4, 3, 7, 5]` |
| 找最大 7（index=4），与首位 1 交换 | swap(0,4) | `[7, 6, 4, 3, 1, 5]` |
| 找最小 1（index=4），与末位 5 交换 | swap(4,5) | `[7, 6, 4, 3, 5, 1]` |

输出：`7 6 4 3 5 1` ✓

**Java 代码：**

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        List<Integer> list = new ArrayList<>();
        while (sc.hasNextInt()) {
            list.add(sc.nextInt());
        }

        int n = list.size();
        int[] arr = list.stream().mapToInt(Integer::intValue).toArray();

        // Step 1: 找最大值，与第一个元素交换
        int maxIdx = 0;
        for (int i = 1; i < n; i++) {
            if (arr[i] > arr[maxIdx]) maxIdx = i;
        }
        if (maxIdx != 0) {
            int tmp = arr[0]; arr[0] = arr[maxIdx]; arr[maxIdx] = tmp;
        }

        // Step 2: 在更新后的数组中找最小值，与最后一个元素交换
        int minIdx = 0;
        for (int i = 1; i < n; i++) {
            if (arr[i] < arr[minIdx]) minIdx = i;
        }
        if (minIdx != n - 1) {
            int tmp = arr[n - 1]; arr[n - 1] = arr[minIdx]; arr[minIdx] = tmp;
        }

        StringJoiner sj = new StringJoiner(" ");
        for (int x : arr) sj.add(String.valueOf(x));
        System.out.println(sj);
    }
}
```

**特殊情况处理：**

| 情况 | 说明 |
|------|------|
| 最大值已在首位 | `maxIdx == 0`，跳过第一次交换 |
| 最小值已在末位 | `minIdx == n-1`，跳过第二次交换 |
| 最小值原在首位 | 第一步交换后，最小值移到 maxIdx 处，第二步可正确找到 |
| 所有元素相等 | 两步均跳过，输出原数组 |

### 题目8：SELECT 与 HAVING 同时使用的是什么？

**答案：`GROUP BY`**

**解析：**

`HAVING` 必须与 `GROUP BY` 搭配使用，用于对**分组后的结果**进行过滤。

**SQL 标准执行顺序：**

```
FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY
```

| 子句 | 作用 | 能否使用聚合函数 |
|------|------|----------------|
| `WHERE` | 对**行**进行过滤（分组前） | ❌ 不能 |
| `GROUP BY` | 按指定列分组 | - |
| `HAVING` | 对**分组结果**进行过滤（分组后） | ✅ 可以 |

**示例：**

```sql
-- 查询平均工资大于 8000 的部门及其平均工资
SELECT department_id, AVG(salary) AS avg_salary
FROM employees
GROUP BY department_id
HAVING AVG(salary) > 8000;
```

```sql
-- 查询员工数量超过 5 人的部门
SELECT department_id, COUNT(*) AS emp_count
FROM employees
GROUP BY department_id
HAVING COUNT(*) > 5;
```

**WHERE vs HAVING 的核心区别：**

```sql
-- WHERE：先过滤行，再分组（不能用聚合函数）
SELECT department_id, AVG(salary)
FROM employees
WHERE salary > 3000          -- 先筛掉工资≤3000的员工
GROUP BY department_id;

-- HAVING：先分组，再过滤组（可以用聚合函数）
SELECT department_id, AVG(salary)
FROM employees
GROUP BY department_id
HAVING AVG(salary) > 8000;  -- 筛掉平均工资≤8000的部门
```

**记忆口诀：**
- `WHERE` → 过滤**行**，在 `GROUP BY` **之前**执行，不能用聚合函数
- `HAVING` → 过滤**组**，在 `GROUP BY` **之后**执行，可以用聚合函数
